Hard Constraints

These rules are absolute. Every agent referencing this document must treat these as inviolable.

1. DATA BOUNDARY: All book intelligence is derived from EXTERNAL PUBLIC SOURCES. Bookmarked does NOT read, scan, or process actual book text. The Content Scanner (Layer 3) is OUT OF SCOPE.

2. LANGUAGE BOUNDARY: Never imply Bookmarked has read the book. Use "external sources report" — never "the book contains" or "what's in this book."

3. NOT A RATING SYSTEM: We surface intelligence so humans can decide. Framing as "ratings" risks being perceived as censorship.

4. NOT A REPLACEMENT FOR LIBRARIANS: Librarians make judgment calls. BI provides the data infrastructure they never had.

5. NOT A CENSOR: We provide data for districts to make defensible decisions within their regulatory framework.

6. ANTI-GAMING: Community actions are transparent, attributed, and additive — never subtractive. Community input adds a layer alongside AI-generated data; it never overwrites or removes the baseline. Administrators always see the same data as librarians. Cross-district patterns expose outliers.

District-Level Ground Truth — Bookmarked's Competitive Moat

Bookmarked connects to actual library systems. When an external source claims a ban and that district is a customer, we verify against ground truth:

  1. External source claims "Book X is banned in District Y"
  2. Bookmarked connects to District Y's library system
  3. Book X is present on the shelf in 14 of 19 libraries
  4. Librarian has marked it "Reviewed — Retained"
  5. Bookmarked reports: "Claimed banned. Bookmarked Verified: Present in 14 libraries. Reviewed and retained."

At 150 districts: meaningful coverage. At 500: definitive national authority.

Bookmarked becomes the source of truth for actual bans and challenges. Everyone else reports claims. Bookmarked reports verifiable facts.

Part 10: Trust, Confidence & Anti-Gaming DESIGNED

Anti-Gaming — Five Principles

  1. Transparent: Every action tied to a named user at a named district
  2. Attributed: Visible to administrators — cannot suppress info from superintendents
  3. Additive: Adds a layer, never overwrites AI baseline
  4. Cross-district exposure: Outlier behavior detectable across the network
  5. Threshold-based: Single actions don't move the needle — multiple independent districts required

Part 12: Community Intelligence PLANNED

Every librarian in 150+ districts becomes a sensor for accuracy, source quality, and ground truth.

Phase 1 — Ship First

Verification/Dispute Layer + Accuracy Feedback

Verification: Every AI-generated field gets Verify (one click) and Dispute (click + correction). Always additive.

Accuracy Feedback: Thumbs up/down per field. Measurable accuracy metrics without requiring correction details.

"Violence: Moderate (AI-Generated) — Verified by 12 districts. 1 dispute: 'Should be Mild.'"

Phase 2 — After Adoption

Source Contribution + Source Unreliable Reporting

Source Contribution: "Submit a source" button — URL classified, validated, ingested.

Source Unreliable: Structured dropdown reasons. Reports add credibility signals, never remove sources. Multiple districts required for downweighting.

Phase 3 — At Scale

District-Level Ground Truth

Bookmarked as verification authority. Connected library systems confirm or contradict external claims.

At 150: start. At 500: definitive. See Part 6.
