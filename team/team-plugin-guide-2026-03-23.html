What changed?

We packaged the 24 skills you use most into a single plugin you can install in any Claude Code project. Your skills now pull brand standards and company context from Confluence instead of local files — so everyone gets the same standards, no matter where they're working.

Nothing about how you use skills changed. Same slash commands, same triggers, same output. Just cleaner plumbing underneath.

Things You Should Know

Lines You Can't Cross

  • Never say we replace librarians. We give them their mission back.
  • Never lead with compliance fear. Lead with library vitality.
  • Never commit dates without engineering. Product timelines need Raju.
  • Never mix district data. District A can't see District B's information.
  • Content Scanner is future only. It's in design, not in the product. Don't tell districts about it.

The BI Language Rule

Book Intelligence describes what is publicly known about books — metadata, reviews, awards, ratings, news from public sources. Never say "what's in the book." Bookmarked does not read book contents. The phrase is "about," not "in."

Product Hierarchy

Bookmarked (the company) → OnShelf (the product platform) → Book Intelligence (the AI engine that powers everything) → Modules (BI Reports, OnShelf Review, Classroom Libraries, OnShelf Engage). Book Intelligence is the engine, not a module.

Brand Standards Come from Confluence

Every skill loads the Brand Design System and Messaging Standards from Confluence before producing output. This means everyone works from the same playbook — colors, fonts, approved phrases, banned phrases, messaging hierarchy. You don't need to memorize them. The skills handle it.

How to Change Things

The plugin is read-only for the team. If you need a skill updated, a brand standard changed, or a new workflow built, use /feedback to submit the request. Claude will help you write it up, and Steve reviews it. This keeps the system stable while making sure good ideas get through.

If something feels broken — a skill gives you wrong output, a boundary seems outdated, a tool isn't connecting — flag it the same way. The faster issues surface, the faster they get fixed.
